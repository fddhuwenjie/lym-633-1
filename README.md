# 志愿者活动招募与工时认证系统

组织者发布活动、岗位、人数和服务时段，志愿者报名后由负责人确认到岗并登记服务时长，支持从报名到证书生成的完整工时认证闭环。

## 快速启动

```bash
# 安装依赖
npm install

# 启动开发服务器（默认 http://localhost:3000）
npm run dev

# 代码检查
npm run lint

# 类型检查
npx tsc --noEmit

# 生产构建
npm run build
```

## 核心功能入口

### 前台（志愿者/负责人/组织者通用）

| 页面 | 路由 | 说明 |
|------|------|------|
| 活动看板 | `/` | 浏览全部活动，按状态筛选，查看岗位余量 |
| 活动详情 | `/activity/:id` | 查看岗位、报名、候补、签到签退 |
| 个人档案 | `/profile` | 报名记录、工时统计、月度图表、证书列表 |
| 工时导出 | `/export` | 按活动/状态/日期筛选，导出 CSV，查看导出历史 |

### 管理后台（组织者/负责人可见，导航栏显示"管理后台"入口）

| 页面 | 路由 | 说明 |
|------|------|------|
| 数据概览 | `/admin` | 活动总数、志愿者数、已认证工时、签发证书 |
| 活动管理 | `/admin/activities` | 创建/编辑/发布活动，管理岗位 |
| 创建活动 | `/admin/activities/new` | 填写活动信息、添加岗位与人数 |
| 报名管理 | `/admin/registrations` | 审核报名、确认录取、候补递补 |
| 工时审核 | `/admin/workhours` | 通过/退回工时，通过后自动生成证书 |
| 证书管理 | `/admin/certificates` | 查看所有证书、撤销证书 |

## 正常业务流程

```
组织者发布活动 → 志愿者报名（满员则候补）
→ 负责人确认录取 → 志愿者签到 → 志愿者签退
→ 负责人审核工时 → 自动生成证书 → 志愿者导出工时
```

- **报名**：选择岗位提交报名，名额已满自动进入候补队列
- **候补递补**：已确认报名取消后，候补队列按顺序自动前移
- **签到签退**：负责人在活动详情页操作，签退后自动计算工时
- **工时审核**：负责人在管理后台通过或退回，退回需填写原因
- **退回修正**：工时被退回后，志愿者可修改后重新提交
- **证书生成**：工时通过后自动生成，编号格式 `VOL-{年份}-{6位序号}`
- **证书撤销**：管理员可撤销证书，编号不回收

## 拦截规则

| 规则 | 触发场景 | 表现 |
|------|---------|------|
| 岗位条件拦截 | 志愿者技能不满足岗位要求 | 报名按钮禁用 |
| 时段冲突拦截 | 同一时段已报名其他活动 | 提交时弹窗提示 |
| 签退时间拦截 | 签退时间早于签到时间 | 提交时提示错误 |
| 自审核拦截 | 负责人审核自己的工时 | 操作列显示"不能自审"，按钮隐藏 |
| 活动结束拦截 | 活动已结束仍尝试报名 | 报名按钮不显示 |

## 数据持久化

所有数据通过 LocalStorage 持久化（前缀 `volunteer_app_`），浏览器刷新或重启后可复查：

- 报名状态与候补顺序
- 签到签退记录
- 工时审核状态
- 证书编号与生成时间
- 导出历史记录

## 验收流程

### 1. 岗位人数释放

1. 以志愿者身份报名某岗位（如"摄影记录"），该岗位满员后进入候补
2. 以另一个已录取志愿者身份取消报名
3. 回到活动详情，确认候补人员自动递补为"待确认"状态
4. 岗位余量正确更新

### 2. 工时退回后重新提交

1. 以负责人身份在管理后台退回某条待审核工时（需填写原因）
2. 以志愿者身份进入个人档案，查看工时状态变为"已退回"
3. 修正后重新提交，状态变为"待审核"
4. 负责人再次审核通过，证书自动生成

### 3. 证书撤销

1. 以负责人身份进入管理后台证书管理页面
2. 点击某条有效证书的"撤销"按钮并确认
3. 证书状态变为"已撤销"，有效证书数减少，已撤销数增加
4. 证书编号不回收，仍可查看历史记录
